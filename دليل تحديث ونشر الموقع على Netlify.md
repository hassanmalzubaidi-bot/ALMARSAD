# دليل تحديث ونشر «مرصد الشرق الأوسط» على Netlify

مستند توثيقي للخطوات التي نُفِّذت فعلياً لتحديث النسخة الحيّة من الموقع، مع المتطلبات والمزالق وحلولها، وموجَّه جاهز لإعادة التنفيذ.

**آخر تحديث ناجح:** نُشر عند 2:14 مساءً — 22 يوليو 2026.
**الرابط الحيّ:** https://famous-biscochitos-e29381.netlify.app
**مشروع Netlify:** `famous-biscochitos-e29381`

---

## 1. الهدف والمتطلبات

- الموقع بأكمله **ملف واحد مكتفٍ ذاتياً** اسمه `index.html` (كل البيانات والخريطة والمستكشف مضمّنة داخله — لا ملفات خارجية عدا خطوط Google).
- عند تطوير نسخة أحدث من `index.html` في مجلد المشروع، المطلوب **رفعها لتحلّ محلّ النسخة القديمة على نفس الموقع ونفس الرابط** (لا موقع جديد ولا رابط جديد).
- الموقع منشور أصلاً عبر **Netlify Drop** (سحب وإفلات يدوي)، لذا لا يوجد ربط بمستودع Git — التحديث يتمّ برفع الملف يدوياً في كل مرة.

## 2. معلومات المشروع الثابتة

| البند | القيمة |
|---|---|
| ملف المصدر | `index.html` (ملف واحد) |
| مجلد المشروع | `…\موقع المرصد - نشر محدث` |
| اسم مشروع Netlify | `famous-biscochitos-e29381` |
| صفحة النشر | `https://app.netlify.com/projects/famous-biscochitos-e29381/deploys` |
| الرابط الحيّ | `https://famous-biscochitos-e29381.netlify.app` |
| النشر التلقائي | مُفعّل (كل رفع يُنشَر مباشرة) |

**كيف تتأكد أن التحديث نجح:** قارن حجم الملف بالبايت. النسخة الجديدة يجب أن يطابق حجمها الحيّ حجم ملفك المحلي (مثال هذه المرة: `3,378,937` بايت بدل القديم `3,204,749`).

## 3. المتطلبات المسبقة

- ملف `index.html` المحدّث موجود في مجلد المشروع.
- الوصول إلى مجلد المشروع مُفعّل في الجلسة (حتى أتمكّن من قراءة الملف ورفعه).
- تسجيل الدخول إلى حساب Netlify فعّال في المتصفّح.
- إضافة «Claude in Chrome» متّصلة (للرفع عبر المتصفّح).

## 4. الخطوات التفصيلية (كما نُفِّذت)

1. **تحديد المشروع الحيّ:** فتح صفحة نظرة عامة على مشروع Netlify للتأكد من الرابط وحالة آخر نشر («Last deployed from Netlify Drop»).

2. **فحص بنية النسخة الجديدة:** التأكد أن `index.html` ملف واحد مكتفٍ ذاتياً (لا سكربتات خارجية)، ومقارنة حجمه بالنسخة المنشورة للتأكد أنه أحدث/أكبر.

3. **فتح صفحة النشر:** الانتقال إلى `…/deploys`. هذه الصفحة تطبيق أحادي (SPA) يُبقي **اتصالاً مباشراً مفتوحاً** لتحديث حالة عمليات النشر لحظياً — وهذا يُبقي المتصفّح في حالة «قيد التحميل» ويعطّل أدوات التقاط عناصر الصفحة (انظر المزالق).

4. **تثبيت الصفحة والتقاط حقل الرفع:** فتح الصفحة من جديد ثم — خلال النافذة الزمنية القصيرة بعد الفتح — تعطيل الاتصال المباشر (EventSource / WebSocket) مؤقتاً، والانتظار حتى تظهر منطقة الرفع، ثم تحديد **حقل رفع الملف الواحد** (يقبل `.html/.htm/.zip`). Netlify يدعم صراحةً: «اسحب مجلد المشروع، أو ملف ZIP، أو **ملف HTML واحد** للنشر فوراً».

5. **رفع الملف:** رفع `index.html` مباشرة إلى ذلك الحقل. بما أن النشر التلقائي مفعّل، يبدأ النشر فوراً.

6. **الانتظار دون أي تنقّل:** بعد بدء الرفع تعرض الصفحة صفّاً بحالة «uploading». **ممنوع إغلاق أو تحديث أو مغادرة الصفحة أثناء الرفع** (Netlify نفسه يعرض تحذيراً بذلك، ويضع حاجز «Leave site?»). قاطعتُ الرفع مرّة بالتنقّل فتوقّف — لذا الانتظار الصامت حتى الاكتمال ضروري.

7. **التحقق من الاكتمال:** بعد اكتمال المعالجة يتحوّل الصفّ إلى **«Production — Published»** بوقت جديد. ثم فتح الرابط الحيّ والتأكد أن حجم الملف المقدَّم يطابق ملفك المحلي بالبايت.

## 5. المزالق وحلولها

| المزلق | السبب | الحل |
|---|---|---|
| أدوات التقاط عناصر الصفحة تفشل بخطأ «page still loading» | لوحة Netlify تُبقي اتصالاً مباشراً مفتوحاً فيبقى المتصفّح «قيد التحميل» | فتح الصفحة من جديد والتقاط الحقل في النافذة القصيرة الأولى + تعطيل الاتصال المباشر مؤقتاً |
| رسالة «الرفع قوطِع ولم يكتمل» | تحديث/تنقّل الصفحة أثناء الرفع | عدم التنقّل إطلاقاً حتى ظهور «Published» |
| رسالة مهلة بعد أمر الرفع رغم نجاحه | المهلة تظهر لأن الصفحة مشغولة فعلاً بالرفع | تجاهُل رسالة المهلة، والتحقق من الحالة الحقيقية لاحقاً |
| الواجهة تبقى «uploading» ولا تتحدّث | تعطيل الاتصال المباشر أوقف التحديث اللحظي للواجهة | إعادة تحميل الصفحة **بعد** اكتمال الرفع لرؤية الحالة الحقيقية |
| صفّ قديم عالق بحالة «uploading» في السجل | محاولة رفع سابقة قوطِعت | غير ضار — يُتجاهَل؛ المعتمد هو صفّ «Published» الأحدث |

## 6. موجَّه جاهز لإعادة التنفيذ معي

انسخ النص التالي وأرسله لي متى أردت تحديث الموقع بعد تعديل `index.html`:

> حدّث النسخة الحيّة من موقع «مرصد الشرق الأوسط» على Netlify.
> - ملف المصدر: `index.html` في مجلد «موقع المرصد - نشر محدث» (ملف واحد مكتفٍ ذاتياً).
> - مشروع Netlify: `famous-biscochitos-e29381` — ارفعه على **نفس الموقع ونفس الرابط**، لا تُنشئ موقعاً جديداً.
> - افتح صفحة deploys، التقط حقل رفع ملف HTML واحد، ارفع `index.html`، ثم **انتظر دون أي تنقّل** حتى تظهر «Published».
> - تأكّد بمقارنة حجم الملف الحيّ بالبايت مع ملفي المحلي، وأعطني الرابط ووقت النشر.

## 7. طريقة بديلة (سطر أوامر — اختيارية)

إن رغبت لاحقاً بنشر أسرع تشغّله بنفسك دون متصفّح:

```bash
# لمرة واحدة:
npm install -g netlify-cli
netlify login            # يفتح المتصفّح لتسجيل الدخول

# في كل تحديث (من داخل مجلد المشروع):
netlify deploy --prod --dir . --site famous-biscochitos-e29381
```

ملاحظة: هذه الطريقة تتطلّب تثبيت Netlify CLI وتسجيل دخول لمرة واحدة، وتتعامل مع مفتاح حسابك محلياً على جهازك.
