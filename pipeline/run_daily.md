# إجراء التحديث اليومي — مرصد الشرق الأوسط

هذا الملف هو التعليمات التي يتبعها الروتين المجدول (وكيل Claude) كل يوم.

## الخطوات

**① الجلب**
```bash
python pipeline/fetch_sources.py
```
يكتب `pipeline/raw_items.json` (أخبار آخر 48 ساعة من RSS + GDELT، مُصفّاة بالصلة).

**② التصنيف (خطوة الوكيل — لا سكربت)**
اقرأ `pipeline/raw_items.json`. لكل خبر **حقيقي وذي صلة** بالشرق الأوسط، أنشئ عنصرًا في مصفوفة تُكتب إلى `pipeline/classified.json` بهذه الحقول بالضبط:
```json
{"التاريخ":"YYYY-MM-DD","الدولة":"إيران","المدينة":"طهران",
 "طبيعة":"عسكري / نووي","نطاق":"دولي","خطر":"حرج",
 "الحدث":"جملة عربية واحدة واضحة","التفاصيل":"جملة إضافية أو \"\"","المصدر":"اسم القناة"}
```
قواعد التصنيف:
- **طبيعة**: من {عسكري، سياسي، دبلوماسي، أمني، اقتصادي، إنساني، قانوني، تقني، اجتماعي، بيئي، أخرى} — يمكن دمجها بـ«/» (نووي→عسكري تلقائيًا).
- **نطاق**: إقليمي / دولي / داخلي.
- **خطر**: حرج / مرتفع جداً / مرتفع / متوسط / منخفض — احكم بحسب حدّة الحدث.
- **الدولة**: الكيان الأساسي؛ يمكن التركيب «إيران / الولايات المتحدة».
- **المدينة**: مدينة معروفة إن ذُكرت، وإلا «عام».
- تجاهل الأخبار غير الجيوسياسية (رياضة، ترفيه، منوّعات) والمكرّرة.
- لا تختلق حدثًا: كل عنصر من خبر فعلي في `raw_items.json`.

**③④ الدمج وإعادة البناء**
```bash
python pipeline/merge_rebuild.py          # أضف --dry لمعاينة دون كتابة
```
يدمج بلا تكرار، يضيف الإحداثيات، ويحدّث `index.html` والعدّادات.

**⑤ النشر (تلقائي عبر Git)**
```bash
git add -A && git commit -m "تحديث يومي: {التاريخ}" && git push
```
Netlify يكتشف الدفعة وينشر تلقائيًا على نفس الرابط خلال دقيقة.

## تحقق
- `merge_rebuild.py` يفشل صراحةً إن تلِف JSON أو اختلّ عدد الكيانات — لا نشر عند الخطأ.
- الملفان `raw_items.json` و`classified.json` مؤقتان (في `.gitignore`).
