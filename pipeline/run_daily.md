# إجراء التحديث اليومي — مرصد الشرق الأوسط (نموذج طابور المراجعة)

الروتين المجدول (وكيل Claude) يتبع هذا الإجراء كل يوم. **الأحداث الجديدة لا تُنشر مباشرة** —
تدخل طابور مراجعة خاصًّا (`data/pending.json`) ينتظر اعتماد الأدمن.

## الخطوات

**① الجلب**
```bash
python pipeline/fetch_sources.py
```
يكتب `pipeline/raw_items.json` (أخبار آخر 48 ساعة من RSS + GDELT، مُصفّاة بالصلة).

**② التصنيف (خطوة الوكيل)**
اقرأ `pipeline/raw_items.json`. لكل خبر حقيقي وذي صلة، أنشئ عنصرًا في `pipeline/classified.json`:
```json
{"التاريخ":"YYYY-MM-DD","الدولة":"إيران","المدينة":"طهران",
 "طبيعة":"عسكري / نووي","نطاق":"دولي","خطر":"حرج",
 "الحدث":"جملة عربية واحدة","التفاصيل":"جملة أو \"\"","المصدر":"اسم القناة"}
```
القواعد: طبيعة من {عسكري، سياسي، دبلوماسي، أمني، اقتصادي، إنساني، قانوني، تقني، اجتماعي، بيئي} (نووي→عسكري) · نطاق: إقليمي/دولي/داخلي · خطر: حرج/مرتفع جداً/مرتفع/متوسط/منخفض · لا تختلق حدثًا · تجاهل غير الجيوسياسي والمكرّر.

**③ الدمج إلى الطابور**
```bash
python pipeline/merge_rebuild.py
```
يضيف الأحداث الجديدة إلى `data/pending.json` (بلا تكرار مقابل الموقع العام والطابور). **لا يمسّ الموقع العام.** يطبع «أُضيف للطابور: N».

**④ الدفع (يحفظ الطابور فقط)**
```bash
git add -A && git commit -m "طابور يومي: {التاريخ}" && git push
```
هذا يحفظ الطابور في المستودع للمراجعة. الموقع العام لا يتغيّر (الأحداث لم تُعتمد بعد).

## اعتماد الأدمن (يدوي — خارج الروتين)
يراجع الأدمن `data/pending.json`، ويضبط `_حالة` لكل عنصر إلى «معتمد» أو «مرفوض»، ثم:
```bash
python pipeline/promote.py      # المعتمد → public/index.html · المرفوض يُهمَل
git add -A && git commit -m "اعتماد ونشر" && git push
```
عندها فقط تظهر الأحداث للجمهور (Netlify ينشر تلقائيًا من مجلد `public/`).

## البنية
- `public/index.html` — الموقع العام (المعتمد فقط) · مجلد النشر في Netlify.
- `data/pending.json` — طابور المراجعة الخاص (خارج النشر — لا يصله الجمهور).
- الملفان `raw_items.json` و`classified.json` مؤقتان (في `.gitignore`).
