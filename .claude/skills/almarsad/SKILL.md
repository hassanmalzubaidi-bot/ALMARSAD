## المهمة الأساسية: تحديث الأخبار

النموذج **نشر مباشر بلا مراجعة**. نفّذ من مجلد المشروع بالترتيب:

### ① الجلب
```bash
python pipeline/fetch_sources.py
```
يكتب `pipeline/raw_items.json` من RSS (الجزيرة، RT عربي، BBC عربي) + GDELT، مُصفّى بصلة الشرق الأوسط.
GDELT قد يفشل بـ429 من عناوين مشتركة — تابع بما توفّر من RSS.

### ② التصنيف (خطوتك أنت — لا سكربت)
اقرأ `pipeline/raw_items.json`. لكل خبر **حقيقي وذي صلة بالشرق الأوسط فقط**، أنشئ عنصرًا في مصفوفة
JSON تكتبها إلى `pipeline/classified.json` بهذه الحقول **بالضبط**:
```json
{"التاريخ":"YYYY-MM-DD","الدولة":"إيران","المدينة":"طهران",
 "طبيعة":"عسكري / نووي","نطاق":"دولي","خطر":"حرج",
 "الحدث":"جملة عربية واحدة واضحة","التفاصيل":"جملة إضافية أو \"\"","المصدر":"اسم القناة"}
```
**قواعد التصنيف (حرجة للجودة):**
- **طبيعة**: من {عسكري، سياسي، دبلوماسي، أمني، اقتصادي، إنساني، قانوني، تقني، اجتماعي، بيئي} — يمكن الدمج بـ«/». النووي يُصنّف **عسكري**.
- **نطاق**: `إقليمي` أو `دولي` أو `داخلي`.
- **خطر**: `حرج` / `مرتفع جداً` / `مرتفع` / `متوسط` / `منخفض` — احكم بحدّة الحدث.
- **الدولة**: الكيان الأساسي؛ يُسمح بالتركيب «إيران / الولايات المتحدة» أو الفاعلين «حماس»، «حزب الله»، «الحوثيون».
- **المدينة**: مدينة معروفة إن ذُكرت، وإلا `عام` أو `دولي`. المدن غير المعروفة أضِف إحداثياتها إلى `CITY_COORDS` في `pipeline/config.py`.
- **لا تختلق حدثًا أبدًا**: كل عنصر من خبر فعلي في `raw_items.json`.
- **تجاهل** غير الجيوسياسي (رياضة، فن، منوّعات، حوادث فردية، إعلانات) والمكرّر.
- اكتب `الحدث` بصياغة تحليلية مركّزة لا نسخًا حرفيًا للعنوان.

### ③④ الدمج والنشر المباشر
```bash
python pipeline/merge_rebuild.py     # يطبع «نُشر: N»
```
يضيف الجديد مباشرةً إلى `public/index.html` (إزالة تكرار ضبابية مقابل القاعدة)، ويحدّث العدّادات.
راجع «إزالة التكرار» أدناه إن ظهر «مكرر مُتجاوَز» عالٍ.

### ⑤ الدفع (ينشر تلقائيًا)
```bash
git add -A && git commit -m "تحديث إخباري: N حدثًا ليوم YYYY-MM-DD" && git push
```

### ⑥ التحقق الحيّ (إلزامي)
انتظر ~20-60 ثانية ثم تحقق أن النشر وصل:
```bash
curl -s "https://famous-biscochitos-e29381.netlify.app/?v=$(date +%s)" -o /tmp/live.html
python -c "import re,json;h=open('/tmp/live.html',encoding='utf-8').read();d=json.loads(re.search(r'<script id=\"EV\" type=\"application/json\">(.*?)</script>',h,re.S).group(1));r=d['rows'];print('إجمالي:',len(r),'| آخر تاريخ:',max(x[0] for x in r))"
```
إن لم يتغيّر الإجمالي بعد دقيقتين، افحص لوحة Netlify → Deploys (قد يكون بناء فاشل — انظر «القيود»).

## بنية البيانات (تنسيق الصف المضغوط `EV`)

البيانات في `public/index.html` داخل `<script id="EV" type="application/json">{...}</script>`:
مفاتيح: `ents` (أسماء الكيانات)، `kinds`، `nat` (الفئات)، `sc` (النطاقات)، `rows`، `net`، `generated`.

كل صف مصفوفة بـ 14 عنصرًا:
```
[0] التاريخ YYMMDD   [1] فهرس الكيان في ents   [2] المدينة   [3] عرض   [4] طول
[5] دقّة (1=مدينة، 0=مستوى دولة)   [6] قناع الفئات bitmask على nat   [7] النطاق 0-3
[8] الخطر 0-5 (0=حرج … 4=منخفض، 5=غير محدد)   [9] الحدث   [10] التفاصيل
[11] المصدر prov (0=أساسي، 1=مصدر مفتوح، 2=دفعة ذكاء اصطناعي)   [12] اسم المصدر
[13] تحقّق (0/1)   [14] مميّز ★ (1=بارز، اختياري)
```
سلّم ألوان الخطر (RC): `["#991b1b","#dc2626","#ea580c","#d97706","#16a34a","#64748b"]`.
لا تعدّل هذا التنسيق يدويًا — استخدم السكربتات أو لوحة الأدمن.

## ميزات الواجهة (public/index.html)
- **الأطلس / الخريطة**: تكبير/تصغير بالعجلة واللمس + أزرار (+/−/↻)، سحب، **تجميع نقاط (clustering)** عند التصغير (نقر العنقود يقرّب)، النقاط تبقى بحجم ثابت عبر CSS `--z`، **شريط زمني** يمرّر الأحداث يومًا بيوم + تشغيل ▶.
- **إبراز الأحداث ★**: الحقل `row[14]=1` → شريط ذهبي «أحداث بارزة» أعلى كل الصفحات (حتى 8، الأحدث أولًا).
- **التقرير التحليلي**: رسوم مكدّسة بألوان درجات الخطر (شكل الاتجاه الشهري + الطبائع). داشبورد لا صفحة نصية.

تعديلات الواجهة = رقعة Python دقيقة على `public/index.html` عبر بحث/استبدال نصّي مؤكَّد، ثم اختبار محلي
(`python -m http.server` + متصفّح، **مع `?nocache=` لتجاوز كاش المتصفّح**)، ثم push. rAF لا يعمل في تبويب
خفيّ — استخدم setTimeout في منطق الرسم.

## القيود والمزالق المعروفة (لا تكرّر اكتشافها)
1. **Netlify المجاني يحظر مساهمي Git غير الموثّقين على المستودعات الخاصة** → لهذا المستودع **عام**. لا تجعله خاصًّا إلا مع خطة Pro. أعراض الفشل: «Build blocked: Unrecognized Git contributor».
2. **الروتين السحابي المجدول يفشل في `git push`** إن لم يكن GitHub مربوطًا ببيئة Claude Code السحابية بصلاحية كتابة. الحل الدائم: ربط حساب hassanmalzubaidi-bot عبر claude.ai → إعدادات → Connect GitHub. حتى يتم ذلك، **حدّث الأخبار يدويًا** بالخطوات أعلاه (المسار المحلي موثوق).
3. **الطابور/المراجعة أُلغيت**: النموذج نشر مباشر. لا يوجد `pending.json` ولا `promote.py`.
4. **`pipeline/raw_items.json` و`classified.json` مؤقتان** (في `.gitignore`) — لا تلتزمهما.
5. عند فحص الموقع الحيّ استخدم دائمًا `?param=` عشوائيًا لتجاوز كاش CDN.
